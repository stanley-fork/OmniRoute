### ًںژڑï¸ڈ Beyond the engines â€” output styles, the adaptive dial & per-request control

The 9 engines above shrink what goes **in**. Three more layers shape **how**, **when**, and what comes **out**:

- **ًںھ„ Output Styles** *(output-axis steering)* â€” inject deterministic, cache-safe response-shaping instructions; combinable, each at `lite` / `full` / `ultra` intensity. Adding a style is a one-line registry entry:

- **ًںژ¯ Adaptive context-budget** *(the dial)* â€” instead of one on/off token threshold, escalate the cheapest, most-lossless engines only as far as needed to **fit the model's context window**. Policy: `reserve-output` (default, model-aware) آ· `percentage` آ· `absolute`. Mode: `floor` (guarantee fit) آ· `replace-autotrigger` (your explicit choice wins) آ· `off` (legacy threshold).
- **ًںژ›ï¸ڈ Where compression is decided** *(precedence, high â†’ low)* â€” per-request `x-omniroute-compression` header â€؛ routing-combo override â€؛ active named profile â€؛ adaptive / auto-trigger â€؛ panel default â€؛ off. Auto-trigger by token threshold, flip on the adaptive dial, pin a named profile, set a one-off per request, or assign a pipeline per routing combo â€” whichever fits the workload.
